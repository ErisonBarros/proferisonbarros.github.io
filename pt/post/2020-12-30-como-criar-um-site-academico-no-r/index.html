Como criar um site acadêmico com o R

Jan 5, 2021 3 minutos de leitura r, tutorial
Imagem do Hugo Themes

Que o R possui múltiplas utilidades para análise e visualização de dados de diversos formatos e em diversas áreas é indiscutível. Entretanto, a união dessa linguagem com o Markdown, dando origem ao R Markdown, e aliado ao IDE RStudio o transforma numa interessante ferramenta bastante diversa para a criação de diversas aplicações complementares ao R, como documentos (.docx, .odt, .pdf), apresentações, livros, currículos, e ainda para a criação de sites. Pretendo escrever mais sobre essa união e utilidades num post futuro.

Neste post, vou focar nos passos para a criação de um site acadêmico no R, como esse aqui. Apesar de existir várias formas de criar um site acadêmico, focarei nesse formato:

  1. usando o pacote blogdown
  2. hospedando no github com esse domínio: <username.github.io>
  3. versionando com git
  4. utilizando o RStudio para facilitar o manejo através de um Projeto R (.Rproj)

Todas as informações referentes à criação e configuração do site podem ser consultadas no livro do pacote blogdown: blogdown: Creating Websites with R Markdown, do Yihui Xie.

Outra fonte de informação são os geniais posts da Profa. Alison Hill. Listo eles aqui:

  1. Up & Running with blogdown
  2. A Spoonful of Hugo: Archetypes
  3. A Spoonful of Hugo: The netlify.toml File
  4. A Spoonful of Hugo: Page Bundles
  5. A Spoonful of Hugo: Troubleshooting Your Build
  6. A Spoonful of Hugo: How much Hugo do I need to know?
  7. Unbreak your blogdown site

Ainda é fundamental destacar o post em português do Diego Trindade. Nele é possível ler mais sobre a discussão do porquê criar e manter um site acadêmico, tudo isso em relação à reprodutibilidade científica.

Passos para criação do site

Os passos seguem essa listagem:

  1. GitHub
  2. RStudio, Git e GitHub
  3. blogdown
  4. Edição do site

É fundamental destacar que será necessário ter algum conhecimento de git, controle de versão, markdown e talvez HTML e CSS ajudem, mas não são fundamentais.

1. GitHub

Primeiramente é necessário criar uma conta on-line no GitHub, se já não possui:

1.1 GitHub: https://github.com/

Com a conta do GitHub criada, será necessário criar um repositório. É fundamental, se quiser que o domínio do site seja nesse formato <username.github.io>, que o nome do repositório seja seu nome de usuário, no meu caso mauriciovancine, seguido de .github.io. Também não adicione o arquivo README.md.

1.2 Repositório: mauriciovancine.github.io

2. RStudio, Git e GitHub

O próximo passo é clonar o repositório criado do GitHub para uma pasta no seu computador. Essa tarefa é relativamente simples, e pode ser feito de diversas formas:

2.1 Diretamente do repositório fazendo o donwload pelo brownse

2.2 Pelo terminar com o comando git clone

2.3 Criando um Projeto R, utilizando o RStudio

  1. Criar um repositório no GitHub NO README + NO .gitignore (ainda!)

  2. clonar o repositório - git clone File > New Project > Version Control > Git

  3. RStudio: install.packages(“blogdown”) library(blogdown) new_site(theme = “gcushen/hugo-academic”, sample = TRUE, theme_example = TRUE, empty_dirs = TRUE, to_yaml = TRUE)

Conferir a versão do Hugo https://themes.gohugo.io/academic/

if yes, then do this update_hugo(force = TRUE)
