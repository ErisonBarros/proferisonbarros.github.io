Como criar um site acadêmico no R

Dec 30, 2020 3 minutos de leitura r, tutorial
Imagem do Hugo Themes

Que o R possui múltiplas utilidades para análise e visualização de dados de diversos formatos e em diversas áreas é indiscutível. Entretanto, a união dessa linguagem com o Markdown, dando origem ao RMarkdown, aliada ao IDE RStudio ainda pode ser novidade para muitas pessoas. Essa junção permite diversas possibilidades de criação de aplicações complementares ao R. Pretendo escrever mais sobre essa união e utilidades num post futuro.

Neste post, vou focar nos passos para a criação de um site acadêmico no R, uma forma bem interessante que achei de desafiar meu conhecimento até então, e principalmente minha paciência…

Existem várias formas de criar um site acadêmico, mas focarei aqui no formato que consegui fazer de forma mais rápida e tranquila, usando o pacote blogdown, hospedando no github, com esse formato: <username.github.io>, git, e o RStudio.

Todas as informações referentes à criação e onfiguração do site podem ser consultadas no livro do pacote blogdown: Creating Websites with R Markdown, do Yihui Xie.

Outra fonte de informação são os geniais posts da Profa. Alison Hill. Deixo eles listados em destaque aqui:

  1. Up & Running with blogdown
  2. A Spoonful of Hugo: Archetypes
  3. A Spoonful of Hugo: The netlify.toml File
  4. A Spoonful of Hugo: Page Bundles
  5. A Spoonful of Hugo: Troubleshooting Your Build
  6. A Spoonful of Hugo: How much Hugo do I need to know?
  7. Unbreak your blogdown site

Fundamental destacar que só encontrei um post em português do Diego Trindade. Nele é possível ler mais informações e também uma discussão muito legal sobro o porquê criar e manter um site acadêmico.

Passos para criação do site

Os passos à seguir seguem essa listagem:

  1. GitHub
  2. RStudio
  3. Git
  4. blogdown
  5. Edição do site

1. GitHub

Primeiramente é necessário criar duas contas on-line:

  1. GitHub: https://github.com/
  2. Criar um repositório no GitHub

Com a conta do GitHub feita, será necessário criar um repositório. Eu criei com o nome de “mauriciovancine2”, só de exemplo, mas pode ser qualquer outro.

  1. Criar um repositório no GitHub NO README + NO .gitignore (ainda!)

  2. clonar o repositório - git clone File > New Project > Version Control > Git

  3. RStudio: install.packages(“blogdown”) library(blogdown) new_site(theme = “gcushen/hugo-academic”, sample = TRUE, theme_example = TRUE, empty_dirs = TRUE, to_yaml = TRUE)

Conferir a versão do Hugo https://themes.gohugo.io/academic/

is your version less than the minimum version? hugo_version()

if yes, then do this update_hugo(force = TRUE)

  1. Netlify segure e arraste a pasta public/ para https://app.netlify.com/drop

  2. criar um arquivo “netlify.toml” file.create(“netlify.toml”)

Open, then copy & paste:
