Contextualização

Que o R possui múltiplas utilidades para análise e visualização de dados de diversos formatos e em diversas áreas é indiscutível. Entretanto, a união dessa linguagem com o Markdown, dando origem ao R Markdown, e aliado ao IDE RStudio, o transforma numa ferramenta bastante versátil para a criação de diversas aplicações complementares ao R, como documentos, apresentações, livros, currículos, e ainda para a criação de sites. Pretendo escrever mais sobre essa união e utilidades num post futuro.

Neste post, vou focar nos passos para a criação de um site acadêmico no R, como esse aqui. Apesar de existir várias formas de criar um site acadêmico, focarei nesse formato:

  1. utilizando o RStudio para facilitar o manejo através de um Projeto R (.Rproj)
  2. usando o pacote blogdown
  3. com o tema acadêmico do HUGO
  4. hospedando no github, com esse domínio: <username.github.io>
  5. versionando com git

Mais informações

Todas as informações referentes à criação e configuração do site pelo R podem ser consultadas no livro do pacote blogdown: blogdown: Creating Websites with R Markdown, do Yihui Xie.

Outra fonte de informação são os geniais posts da Profa. Alison Hill. O mais recente deles trás uma atualização do processo de criação do site utilizando a nova versão do tema acadêmico do HUGO, e pode ser conferido aqui: Up & running with blogdown in 2021.

Ainda é fundamental destacar o post Blogdown Tutorial (in Portuguese) do Diego Trindade. Nele é possível ler mais sobre a discussão do porquê criar e manter um site acadêmico, principalmente em relação à reprodutibilidade científica.

É fundamental destacar que é recomendável ter conhecimento de git, controle de versão, markdown e talvez HTML e CSS ajudem, mas não são fundamentais.

Também é fundamental que estejam instalados o R, RStudio e git em seu computador, e de preferência, com o RStudio sincronizado com o GitHub. Para fazer isso, veja esse capítulo de livro da .

1. Pré-site

1.1 GitHub

Primeiramente é necessário criar uma conta on-line no GitHub, se já não a possui:

1.1 GitHub: https://github.com/

Com a conta do GitHub criada, será necessário criar um repositório.

Se quiser que o domínio do site seja nesse formato <username.github.io>, é necessário que o nome do repositório seja seu nome de usuário, no meu caso mauriciovancine, seguido de .github.io. Também não adicione o arquivo README.md ou .gitignore.

1.2 Repositório: mauriciovancine.github.io

2. RStudio e git

O próximo passo é clonar o repositório criado do GitHub para uma pasta no seu computador. Essa tarefa é relativamente simples, e pode ser feito de diversas formas:

2.1 Diretamente do repositório fazendo o donwload pelo brownse no formato .zip

2.2 Pelo terminal (se estiver no Linux, MacOS ou utilizando o Git Bash no Windows) com o comando git clone e o formato de clone do repositório, no meu caso https://github.com/mauriciovancine/mauriciovancine.github.io.git

2.3 Criando um Projeto R, utilizando o RStudio (acho esse o caminho mais fácil)

  1. RStudio: install.packages(“blogdown”) library(blogdown) new_site(theme = “gcushen/hugo-academic”, sample = TRUE, theme_example = TRUE, empty_dirs = TRUE, to_yaml = TRUE)
